## 📊 資料庫資訊

這些帳號儲存在：
- **資料庫**：NGOPlatformDb
- **表格**：Workers
- **WorkerId**：28（管理員）、29（員工）、30（督導）

### 查詢測試帳號 SQL

```sql
-- 查詢所有測試帳號
SELECT
    WorkerId,
    Email,
    Name,
    Role,
    CreatedAt
FROM Workers
WHERE Email IN ('admin@ngo.org', 'supervisor@ngo.org', 'staff@ngo.org')
ORDER BY WorkerId;
```

## ➕ 如何創建新測試帳號

使用 API 端點創建新帳號（會自動加密密碼）：

```bash
curl -X POST "http://localhost:5264/api/Worker" \
  -H "Content-Type: application/json; charset=utf-8" \
  -d '{
    "email": "newuser@ngo.org",
    "password": "NewPass123!",
    "name": "新測試用戶",
    "role": "staff"
  }'
```

**角色選項**：
- `admin` - 管理員
- `supervisor` - 督導
- `staff` - 一般員工

## ⚠️ 安全提醒

### 注意事項：

1. **僅供本地測試使用**
   - 這些是測試帳號，不要用於正式環境
   - 密碼設計簡單，方便測試

2. **正式環境建議**
   - 使用複雜密碼
   - 定期更換密碼
   - 啟用雙因素認證（如果有）

3. **刪除測試帳號**
   - 部署到正式環境前，請刪除所有測試帳號
   ```sql
   DELETE FROM Workers
   WHERE Email IN ('admin@ngo.org', 'supervisor@ngo.org', 'staff@ngo.org');
   ```

## 🧪 登入測試步驟

1. 開啟瀏覽器訪問：http://localhost:5173
2. 輸入測試帳號 Email（例如：`admin@ngo.org`）
3. 點擊「下一步」
4. 輸入密碼（例如：`Admin123!`）
5. 點擊「登入」
6. 成功後會跳轉到 Dashboard

## 🔄 重置密碼

如果需要重置測試帳號密碼：

```bash
# 使用 API 更新密碼（需要 WorkerId）
curl -X PUT "http://localhost:5264/api/Worker/{WorkerId}/password" \
  -H "Content-Type: application/json" \
  -d '{
    "newPassword": "NewPassword123!"
  }'
```

或者直接刪除帳號後重新創建。

## 📞 遇到問題？

### 登入失敗常見原因：

1. **Email 輸入錯誤** - 確認是否完全一致（區分大小寫）
2. **密碼輸入錯誤** - 注意大小寫和特殊符號
3. **API 未啟動** - 確認 WebAPI 正在運行（http://localhost:5264）
4. **資料庫連線問題** - 確認 SQL Server 服務已啟動

### 驗證步驟：

```bash
# 測試 API 是否運行
curl http://localhost:5264

# 查詢測試帳號是否存在
# 在 SSMS 執行：
SELECT Email, Name, Role FROM Workers
WHERE Email = 'admin@ngo.org';
```
