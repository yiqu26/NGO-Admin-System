## ⚠️ 常見問題

### Q1: WebAPI 無法連接資料庫
**解決方式**：
1. 確認 SQL Server 服務已啟動
2. 在 SSMS 確認能連線到 `YUNYUE\SQLEXPRESS`
3. 確認資料庫名稱為 `NGOPlatformDb`
4. 檢查 `appsettings.json` 的連線字串是否正確

### Q2: React 無法連接 WebAPI
**檢查清單**：
1. WebAPI 是否已啟動？（必須先啟動 WebAPI）
2. WebAPI 是否在 `http://localhost:5264` 運行？
3. 檢查 `.env.development` 中的 `VITE_API_BASE_URL` 是否正確
4. 檢查瀏覽器 Console 是否有 CORS 錯誤

### Q3: CORS 跨域錯誤
**需要在 WebAPI 的 Program.cs 添加 CORS 設定**：
```csharp
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowReactApp", policy =>
    {
        policy.WithOrigins("http://localhost:5173")
              .AllowAnyHeader()
              .AllowAnyMethod();
    });
});

// 在 app.Use... 之前添加
app.UseCors("AllowReactApp");
```

### Q4: Azure 相關功能無法使用
**說明**：
本地開發環境暫未配置以下 Azure 服務（功能受限但不影響基本運作）：
- Azure Blob Storage（檔案上傳功能）
- Azure Speech（語音轉文字功能）
- Azure OpenAI（AI 輔助功能）

如需使用這些功能，請在 `appsettings.json` 中填入對應的 Azure 服務金鑰。

## 📝 開發注意事項

1. **環境變數優先級**：
   - `.env.development` 用於本地開發
   - `.env.production` 用於正式環境
   - Vite 會自動根據執行模式載入對應檔案

2. **資料庫變更**：
   - 如果資料庫結構有變更，需執行 Entity Framework Migration
   - `dotnet ef migrations add [MigrationName]`
   - `dotnet ef database update`

3. **Port 衝突**：
   - 如果預設 port 被占用，可在 `launchSettings.json` 修改
